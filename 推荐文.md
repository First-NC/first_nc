# 写了几百行 G 代码，你敢直接上机吗？

操机最怕什么？不是撞刀，是程序错到最后一刀才发现。

一块多层实木板，正反面各跑一套 NC 程序。正面 XYZ 刻完了，翻面 UVW 走反面。文件一换，坐标系一乱——整块板材报废。

以前怎么校验？打印代码，荧光笔逐行比对。或者装个仿真软件，Windows 独占，Mac 和 Linux 用户先装虚拟机。再或者，硬着头皮直接上机，开空跑盯着刀尖看。

**First NC 让这件事变得简单了。**

![First NC 三面板联动界面：左侧文件列表、中间 Monaco 代码编辑器、右侧 3D 刀路仿真，同屏协作](https://cdn.jsdelivr.net/gh/First-NC/first_nc@main/screenshots/02_three_panel.png)

一个窗口，三块面板：左侧文件列表、中间代码编辑器、右侧 3D 视图。改一行 `G01`，仿真秒级刷新。点一下 3D 路径，编辑器自动跳到对应行。**编辑器和仿真双向联动，所见即所得。**

## 核心卖点：激光双面加工「同屏可视化」

这是 First NC 和其他 NC 工具最大的区别。

双面加工程序包含两套坐标系——正面 `XYZ`、反面 `UVW`。传统做法是开两个文件或两个窗口来回切，操作员凭记忆比对刀路。

First NC 的做法是：**正反面刀路渲染在同一张 3D 场景里。**

![激光模式下双坐标系同屏渲染：UVW 反面路径以紫色独立着色，与正面青色路径清晰区分。图例中单独标注「UVW 反面路径」](https://cdn.jsdelivr.net/gh/First-NC/first_nc@main/screenshots/laser.png)

- **自动识别**：打开文件自动扫描 `U/V/W` 指令字，有则切到「激光模式」，无则保持「雕刻机模式」
- **独立着色**：UVW 反面路径用紫色（#a855f7）渲染，图例标注「UVW 反面路径」
- **全联动**：模式切换后，编辑器、进度条、3D 拾取、倍速播放全部同步工作

切换到沉浸式 3D 全屏后，刀路展示更专注，双面加工路径的层次感也更直观：

![沉浸式 3D 全屏模式：聚焦刀路仿真，正反面刀路层次分明](https://cdn.jsdelivr.net/gh/First-NC/first_nc@main/screenshots/06_immersive_3d.png)

## 一个完整的仿真播放器

不只是静态看图，First NC 提供完整的播放控制系统——就像看视频一样看刀路跑完全程。

![仿真播放器运行中：播放/暂停、单步、速度档位、进度条拖拽、跟随刀位](https://cdn.jsdelivr.net/gh/First-NC/first_nc@main/screenshots/05_playback.png)

- **播放 / 暂停** —— 像看视频一样看刀路走完
- **单步前进 / 后退** —— 逐行验证关键段
- **三档速度** —— 低速细看、标准播放、快速过一遍

## 开箱即用，6MB 起步

安装包极小，不存在「下个软件先等半小时」的情况：

| 平台 | 安装包大小 |
|---|---|
| Windows | **6.8 MB**（EXE / MSI） |
| macOS | **9.2 MB**（DMG） |
| Linux | **8.8 MB**（DEB） |

相比 Electron 套壳动辄两三百 MB 的体量，Tauri 架构的优势在安装包大小上体现得最直观。而且三平台都有原生安装包，**一套代码编译 Windows、Linux、macOS 三个版本。**

从 v1.0.0 到 v1.2.5 已经迭代了 **30 个版本**，代码库 85 个 TypeScript 模块、692 KB 前端源码。功能性、稳定性经过近半年的持续打磨。

**其他值得一提的能力：**

- 原生 `.nc` / `.anc` 文件支持，自动检测 UTF-8 / GBK 编码
- 四种主题（浅色 / 藏蓝 / 深色 / 跟随系统），车间强光或暗光都能用
- 中英文一键切换，默认中文界面
- Monaco 编辑器加持（VS Code 同款引擎），代码折叠、语法高亮、搜索替换全有
- 最近 10 个文件自动恢复
- 应用内增量更新，有新版自动提示安装
