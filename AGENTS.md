# first_nc 协作细则

本文件适用于 `first_nc/`。

## 项目定位

`first_nc` 是桌面端主应用，当前技术栈为 `React + Vite + TypeScript + Tauri`。
目录中同时包含前端界面、桌面端能力、打包脚本与发布相关内容。

## 开发与验证

- 安装依赖：`npm install`
- 本地开发：`npm run dev`
- 前端构建：`npm run build`
- Tauri 开发：`npm run tauri:dev`
- Tauri 构建：`npm run tauri:build`
- 打包相关测试：`npm run test:packaging`
- Lint：`npm run lint`

修改完成后，至少执行与改动直接相关的命令。
如果改动影响打包、版本同步或桌面端行为，优先补跑对应脚本或测试。

## 编码要求

- 组件层优先负责展示与交互，不要把过多业务规则直接堆进 JSX 或大型组件中。
- 编辑器、3D 视图、文件处理、播放控制、国际化、桌面端桥接应尽量保持职责分离。
- 如果逻辑已具备复用价值，优先抽成独立模块或 hooks，而不是复制到多个组件。
- 不要把与 `Tauri` 强绑定的逻辑扩散到普通展示组件里。

## 重点目录

- `src/`：前端主代码
- `src-tauri/`：桌面端相关代码与配置
- `scripts/`：版本同步、打包与流程脚本

## 修改注意事项

- 修改 `scripts/`、`src-tauri/`、版本号同步逻辑时，必须控制影响范围。
- 优先复用现有脚本和命名方式，不要引入风格完全不同的新流程。
- 涉及 UI 改动时，避免把接口适配、复杂计算、状态编排全部压进页面组件。
- 涉及 3D 视图与编辑器联动时，要特别注意数据流是否清晰，避免形成隐式耦合。
